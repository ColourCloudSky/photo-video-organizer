# 照片视频整理

## 概览

> 作者：彩云sky ｜ 关注公众号「彩云译设计」获取更多 AI + 设计实战内容

当用户需要整理本地照片、视频、外置硬盘相册或手机导出的媒体文件时，使用这个 skill。优先使用内置脚本执行，因为脚本支持干跑预览、按文件创建时间归档、创建时间缺失时回退到修改时间、跳过已经正确归档的文件，并且不会覆盖已有文件。

## 默认流程

1. 确认用户给出的根目录，以及用户想做哪一种操作：
   - 只归档：把图片和视频移动到年份/月目录。
   - 只改名：在当前位置按时间戳重命名媒体文件。
   - 先归档再改名：适合混乱的大型相册库。
2. 除非用户明确要求直接执行，否则先运行 `--dry-run` 预览。
3. 向用户汇总关键数量：媒体总数、已经在正确位置的数量、需要移动的数量、可能重复的数量、需要改名的数量。
4. 在用户确认后执行实际操作；如果用户已经明确要求“帮我整理/移动/改名”，可以直接执行，但要遵守权限审批。
5. 执行后再次验证：
   - 归档任务：确认还需要移动的媒体数量为 0。
   - 改名任务：确认所有媒体文件名都符合 `YYYYMMDD.ext` 或 `YYYYMMDD-编号.ext`。

## 目录规则

- 默认归档目标是 `<根目录>/_YYYY/M月/文件名`。
- 示例：`/Volumes/sky/相册/xx/_2026/1月/20260120.HEIC`。
- 保留已有目录格式，不要把 `_2026` 改成 `2026`，也不要把 `1月` 改成 `01月`。
- 如果目标月份文件夹不存在，就创建它。
- 如果媒体文件已经在正确的目标月份文件夹里，就跳过，不要重复移动。
- 只整理最终的图片和视频文件，不要重构无关文件夹，也不要移动普通文档。

## 命名规则

- 改名格式为 `YYYYMMDD.ext`，保留原始扩展名和扩展名大小写。
- 如果同一个文件夹里同一天、同扩展名有多个文件，使用递增编号：
  - `20250501.HEIC`
  - `20250501-001.HEIC`
  - `20250501-002.HEIC`
- 大批量改名时必须使用两步改名，先改成临时名，再改成最终日期名，避免文件名互相占用或覆盖。

## 重复与同名处理

- 永远不要覆盖已有文件。
- 如果目标位置已经有同名文件，并且两个文件字节完全一致，只有在用户要求“不要重复”或使用 `--dedupe-identical` 时，才删除源文件并保留目标位置那一份。
- 如果目标位置有同名文件但内容不同，必须保留两个文件，并给新移入的文件追加编号后缀。
- 如果存在 macOS AppleDouble 伴随文件，例如 `._filename`，在启用 `--include-sidecars` 时让它跟随对应媒体一起移动/改名。
- 不要把 `._filename` 伴随文件当成照片或视频本体。

## 支持的媒体类型

默认支持这些扩展名：

`jpg`, `jpeg`, `png`, `heic`, `heif`, `tif`, `tiff`, `webp`, `gif`, `bmp`, `mp4`, `mov`, `m4v`, `avi`, `3gp`, `mts`, `mkv`

不要默认处理 `.aae` 文件；只有用户明确要求处理 iPhone 编辑记录时再讨论。

## 内置脚本

脚本位置：

```bash
scripts/organize_media.py
```

常用命令：

```bash
# 预览：按 _年份/月月 归档，不实际移动
python3 scripts/organize_media.py --root "/Volumes/sky/相册/xx" --archive --dry-run

# 执行：按 _年份/月月 归档，并去掉目标位置已有的完全相同重复文件
python3 scripts/organize_media.py --root "/Volumes/sky/相册/xx" --archive --dedupe-identical

# 预览：在当前位置按日期改名，不实际改名
python3 scripts/organize_media.py --root "/Volumes/sky/相册/xx" --rename --dry-run

# 执行：在当前位置按日期改名
python3 scripts/organize_media.py --root "/Volumes/sky/相册/xx" --rename

# 执行：先归档，再按日期改名
python3 scripts/organize_media.py --root "/Volumes/sky/相册/xx" --archive --rename --dedupe-identical
```

参数说明：

- `--root PATH`：必填，媒体库根目录。
- `--archive`：把媒体移动到年份/月目录。
- `--rename`：把媒体文件按时间戳重命名。
- `--dry-run`：只预览，不修改文件。
- `--dedupe-identical`：只有当目标位置已经存在字节完全一致的文件时，才删除源文件。
- `--year-prefix "_"`：年份目录前缀，默认创建 `_2026`；如果要创建 `2026`，传空字符串。
- `--month-suffix "月"`：月份目录后缀，默认创建 `1月`。
- `--include-sidecars`：让 `._filename` 伴随文件跟随媒体一起移动/改名。

## 安全注意事项

- 外置硬盘读取元数据和逐字节比较会比较慢，长任务要持续给用户简短进度更新。
- 如果目标目录在工作区之外，实际移动、删除、改名之前要按权限流程请求批准。
- 不要擅自删除重复文件；只有用户明确要求去重，或命令使用了 `--dedupe-identical` 时才做。
- 如果任务中断，可以重新运行同一命令。脚本会跳过已经在正确位置的文件，并使用安全后缀处理同名冲突。
